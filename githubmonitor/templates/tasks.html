关键字设置说明

当前扫描通过 GitHub REST API 的 Search code 接口执行；关键词区域支持批量配置，每行作为一次独立查询。

基础匹配 secret_key 搜索文件内容中的普通关键字。
带空格短语 "hello world" 短语、空格或特殊含义文本建议用双引号包裹。
同时包含 mysql AND password 同时匹配多个词；空格分隔多个词通常也会缩小结果。
排除关键字 mysql NOT localhost 匹配 mysql，但排除包含 localhost 的结果。
限定仓库 password repo:owner/repo 只扫描指定仓库；也可使用 user:owner 或 org:orgname。
限定语言 token language:python 按 GitHub Linguist 识别的语言过滤。
限定路径 password path:config 只匹配指定目录或路径下的代码。
限定文件名 DB_PASSWORD filename:.env 按文件名过滤，适合查找配置文件。
限定扩展名 private_key extension:pem 按文件扩展名过滤，不需要写点号。
限定大小 secret size:<10000 按文件字节大小过滤，可使用 >、< 或区间。
搜索范围 token in:file,path 默认搜索文件内容；可用 in:file、in:path 或 in:file,path。
包含 Fork password fork:true 需要包含符合 GitHub 索引条件的 fork 仓库时使用。
限制 查询必须包含至少一个搜索词；只搜索默认分支、可索引文件和小于 384 KB 的文件。REST 搜索查询超过 256 个字符或包含过多布尔操作符会被 GitHub 拒绝。
符号处理 GitHub legacy code search 会忽略这些符号：. , : ; / \ ` ' " = * ! ? # $ & + ^ | ~ < > ( ) { } [ ] @。如需查找带符号的完整片段，建议配合双引号、文件名、路径、语言等限定词缩小范围。
预览 可在浏览器打开 https://github.com/search?type=code&q=URL编码后的关键字 预览结果；本系统会保存 GitHub API 返回的 text-match 片段。
